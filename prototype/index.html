Building Sayfa: A Static Site Generator in Elixir

Introduction

I've been wanting to build a static site generator in Elixir for a while. The goal was simple: create something extensible, fast, and easy to use. Sayfa (Turkish for "page") is the result.

Here's some inline code and a block of code below. Notice the language label and copy button:

elixir 
defmodule Sayfa.Builder do
  @moduledoc "Core build pipeline orchestration."

  def build(opts \\ []) do
    config = Config.resolve(opts)

    with {:ok, files} <- discover_files(config.content_dir),
         {:ok, contents} <- parse_files(files, config) do
      render_and_write(contents, config)
    end
  end
end

Architecture

Sayfa follows a two-layer architecture. The core package provides reusable SSG functionality, while user sites depend on it via Hex. This keeps the core focused and user sites lightweight.

The best tool is one that gets out of your way and lets you focus on creating content.

Content Pipeline

Content flows through distinct stages, each responsible for a single transformation:

  1. Load — Read files from disk
  2. Parse — Extract front matter + markdown
  3. Transform — Apply hooks, generate slugs
  4. Render — Apply EEx templates
  5. Write — Output to disk

Parsing

The parser uses YamlElixir for front matter and MDEx (a Rust NIF) for markdown rendering. This gives us excellent performance while supporting all CommonMark features.

Rendering

Templates use Elixir's built-in EEx engine. Layouts compose in three layers: content body, layout template, and base template.

Conclusion

Building Sayfa has been a great learning experience. Elixir's pattern matching and pipeline operator make content processing elegant and readable.

OTP Basics Cheat Sheet

June 1, 2024
elixir otp

A quick reference for the most common OTP behaviours and when to use them.

GenServer

Use GenServer when you need a process that maintains state and handles synchronous or asynchronous requests. It's the workhorse of OTP.

elixir 
defmodule MyApp.Counter do
  use GenServer

  def start_link(init), do: GenServer.start_link(__MODULE__, init)
  def increment(pid), do: GenServer.call(pid, :increment)

  @impl true
  def init(count), do: {:ok, count}

  @impl true
  def handle_call(:increment, _from, count), do: {:reply, count + 1, count + 1}
end

Supervisor

Supervisors monitor child processes and restart them according to a strategy. The three main strategies are :one_for_one, :one_for_all, and :rest_for_one.

When to use what

  • GenServer — stateful process, request/response
  • Agent — simple state wrapper (thin GenServer)
  • Task — one-off async work
  • Supervisor — fault tolerance, process monitoring
